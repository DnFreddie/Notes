Automatically Cleaning Up Logs with logrotate

At the end of each rotation period, the log files are renamed and pushed toward the end of the chain of logs as a new log file is created, replacing the current log file. For instance, /var/log.auth will become /var/log.auth.1, then /var/log.auth.2, and so on

Configure

The config is located into /etc/logrotate.conf
